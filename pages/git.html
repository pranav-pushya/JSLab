📋 Git Commands Cheatsheet

Setup

git initInitialize a new repository
git clone <url>Clone a remote repository
git config --global user.nameSet your name
git config --global user.emailSet your email

Basic Workflow

git statusSee changed files
git add <file>Stage a file
git add .Stage all changes
git commit -m "msg"Commit with message
git log --onelineView compact history

Branching

git branchList all branches
git branch <name>Create a new branch
git checkout <branch>Switch to a branch
git checkout -b <name>Create + switch
git merge <branch>Merge branch into current
git branch -d <name>Delete a branch

Remote

git remote add origin <url>Add remote
git push origin mainPush to remote
git pull origin mainPull from remote
git fetchDownload remote changes

Undo / Fix

git diffSee unstaged changes
git stashTemporarily save changes
git stash popRestore stashed changes
git reset <file>Unstage a file
git checkout -- <file>Discard file changes

Info

git logFull commit history
git log --oneline --graphVisual branch graph
git show <hash>Show commit details
git remote -vList remote repos

🎯 Practice Scenarios

Scenario 1: Start a New Project

  1. Create a new repository: git init
  2. Create files: touch index.html style.css
  3. Stage all files: git add .
  4. Make your first commit: git commit -m "Initial commit"
  5. Check your log: git log --oneline

Scenario 2: Feature Branch Workflow

  1. Create a feature branch: git checkout -b feature-nav
  2. Make changes and stage: git add .
  3. Commit: git commit -m "Add navigation bar"
  4. Switch back to main: git checkout main
  5. Merge the feature: git merge feature-nav
  6. Delete the branch: git branch -d feature-nav

Scenario 3: Push to GitHub

  1. Add remote: git remote add origin https://github.com/user/repo.git
  2. Push to GitHub: git push -u origin main
  3. Pull latest changes: git pull origin main
